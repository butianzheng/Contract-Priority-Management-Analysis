# DPM 测试数据说明

## 数据概览

本测试数据集用于验证 DPM 系统的功能和性能，包含：

- **15个客户** （A/B/C三个等级）
- **80条合同** （多种场景覆盖）
- **3个策略** （均衡、客户优先、生产优先）
- **23条工艺难度配置** （5种钢种，多厚度范围）
- **6条节拍标签** （1-3日达配置）
- **10条人工干预记录** （展示Alpha调整）

## 客户分布

### A类客户（VIP）- 4个
- CUST001: 宝钢集团 (AAA)
- CUST002: 首钢集团 (AA)
- CUST003: 鞍钢集团 (AA)
- CUST004: 武钢集团 (AAA)

### B类客户（重点）- 5个
- CUST005-009: 河北钢铁、马钢、沙钢、华菱、建龙

### C类客户（普通）- 6个
- CUST010-015: 日照、永钢、中天、方大、凌钢、通钢

## 性能测试场景

### 场景1：大量数据排序
- 80条合同的优先级计算和排序
- 预期时间：<100ms（单次）

### 场景2：多策略切换
- 在3个策略之间快速切换
- 每次切换需重新计算80条合同
- 预期时间：<150ms（每次切换）

### 场景3：Alpha调整响应
- 修改Alpha系数后立即刷新
- 预期时间：<100ms

### 场景4：历史记录查询
- 查询单个合同的干预历史
- 预期时间：<50ms

## 预期优先级排序逻辑

### 均衡策略（ws=0.5, wp=0.5）
优先级靠前的典型合同：
1. A类客户 + 紧急交期 + 高利润
2. B类客户 + 中等交期 + 高难度
3. C类客户 + 常规订单

### 客户优先策略（ws=0.7, wp=0.3）
优先级靠前的典型合同：
1. A类客户（无论交期）
2. B类客户 + 较高利润
3. C类客户 + 高利润

### 生产优先策略（ws=0.3, wp=0.7）
优先级靠前的典型合同：
1. 高难度材料（304、Q550）
2. 特殊规格族
3. 大厚度产品

## 测试建议

### 功能测试
1. 切换不同策略，观察排序变化
2. 点击合同查看详情
3. 调整Alpha系数，观察优先级变化
4. 查看干预历史记录
5. 观察紧急/警告状态标识

### 性能测试
1. 首次加载80条数据的时间
2. 策略切换响应时间
3. 对话框打开速度
4. Alpha调整后刷新速度
5. 滚动大量数据的流畅度

### 边界测试
1. Alpha最大值（2.0）和最小值（0.0）
2. 交期为0天的紧急订单
3. 同一合同的多次干预记录
4. 带有Alpha标识（★）的合同显示
