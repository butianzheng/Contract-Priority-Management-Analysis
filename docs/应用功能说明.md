# DPM 合同动态优先级管理系统 - 功能说明文档

## 1. 系统概述

DPM (Dynamic Priority Management) 是一个基于 Tauri + React + Rust + SQLite 的跨平台桌面应用，用于合同动态优先级管理。

### 核心功能
- 合同优先级评分引擎（S-Score + P-Score）
- 策略权重动态切换
- 本地 SQLite 数据存储
- 合同分类聚合与表格展示

### 技术栈
- **前端**: React + TypeScript + Vite
- **后端**: Rust（评分引擎 + 数据库服务）
- **数据库**: SQLite（本地存储）
- **框架**: Tauri（桌面应用框架）

## 3. 功能模块详细说明

### 3.1 仪表盘 (Dashboard)

**路径**: `/dashboard`

**功能描述**:
- 系统概览页面，展示关键指标和统计数据
- 显示合同总数、待处理合同数等核心指标
- 提供快速导航入口

**主要组件**:
- 统计卡片区域
- 快捷操作按钮
- 最近活动列表

---

### 3.2 合同管理

#### 3.2.1 合同列表 (Contracts)

**路径**: `/contracts`

**功能描述**:
- 显示所有合同及其优先级计算结果
- 支持按策略选择计算优先级
- 提供多维度筛选功能

**主要功能**:
- 策略选择下拉框：选择不同策略计算优先级
- 筛选面板：按客户、规格族、厚度、宽度等条件筛选
- 合同表格：展示合同详情、S-Score、P-Score、优先级
- 人工干预：支持设置 Alpha 系数调整优先级

**表格字段**:
| 字段 | 说明 |
|------|------|
| 排名 | 当前策略下的优先级排名 |
| 合同编号 | 唯一标识 |
| 客户 | 客户ID |
| 规格族 | 产品规格分类 |
| 厚度 | 产品厚度 (mm) |
| 宽度 | 产品宽度 (mm) |
| 交期 | 计划交货日期 |
| 剩余天数 | 距交期剩余天数 |
| S-Score | 战略价值评分 |
| P-Score | 生产难度评分 |
| 优先级 | 综合优先级分数 |
| Alpha | 人工调整系数 |

#### 3.2.2 变更历史 (ContractHistory)

**路径**: `/contracts/history`

**功能描述**:
- 查看合同的干预历史记录
- 追踪 Alpha 系数调整记录

---

### 3.3 策略沙盘

#### 3.3.1 模拟计算 (Sandbox)

**路径**: `/sandbox`

**功能描述**:
- 模拟不同权重参数对优先级排名的影响
- 支持实时调整 ws（S-Score权重）和 wp（P-Score权重）
- 对比原始排名与模拟排名的变化

**页面布局**:
- **左侧参数面板**（260px 宽度）:
  - 基准策略选择
  - 权重调节滑块（ws、wp）
  - 合同筛选（客户、规格族、紧急合同）
  - 运行模拟按钮

- **右侧结果区域**:
  - 统计卡片：模拟合同数、排名上升、排名下降、平均优先级变化
  - 对比表格/图表视图切换
  - 详细对比数据

**权重调节说明**:
- ws: S-Score 权重，范围 0-1
- wp: P-Score 权重，范围 0-1
- 权重和应等于 1.0（系统会校验）

#### 3.3.2 结果对比 (SandboxCompare)

**路径**: `/sandbox/compare`

**功能描述**:
- 对比两个不同策略的优先级计算结果
- 展示排名变化和优先级差异

**页面布局**:
- **策略选择区**:
  - 策略 A 选择框 + 权重信息显示
  - VS 分隔符
  - 策略 B 选择框 + 权重信息显示
  - 开始对比按钮

- **统计卡片**:
  - 对比合同数
  - 排名上升数量
  - 排名下降数量
  - 排名不变数量
  - 平均排名偏差

- **对比表格**:
  | 字段 | 说明 |
  |------|------|
  | 合同编号 | 合同唯一标识 |
  | 客户 | 客户ID |
  | 规格族 | 产品规格分类 |
  | 策略A排名 | 策略A下的排名 |
  | 策略B排名 | 策略B下的排名 |
  | 排名变化 | ↑/↓/- 表示上升/下降/不变 |
  | 策略A优先级 | 策略A计算的优先级分数 |
  | 策略B优先级 | 策略B计算的优先级分数 |
  | 优先级差异 | 两策略优先级差值 |

#### 3.3.3 策略模板 (StrategyTemplates)

**路径**: `/sandbox/templates`

**功能描述**:
- 管理策略权重模板
- 快速创建预设策略
- 编辑和删除现有策略

**页面布局**:

- **预设模板区**:
  展示 5 个内置预设模板卡片：
  | 模板名称 | ws | wp | 说明 |
  |----------|-----|-----|------|
  | 均衡策略 | 0.5 | 0.5 | S-Score 和 P-Score 权重相等 |
  | 客户优先 | 0.7 | 0.3 | 侧重客户战略价值 |
  | 生产优先 | 0.3 | 0.7 | 侧重生产难度优化 |
  | 极端S策略 | 1.0 | 0.0 | 完全依据客户战略价值 |
  | 极端P策略 | 0.0 | 1.0 | 完全依据生产难度 |

- **当前策略区**:
  - "创建新策略"按钮
  - 策略列表表格：策略名称、ws权重、wp权重、权重和、操作（编辑/删除）

- **创建/编辑模态框**:
  - 策略名称输入框（编辑时禁用）
  - S-Score 权重滑块
  - P-Score 权重滑块
  - 权重和校验显示
  - 保存/取消按钮

## 4. 评分引擎说明

### 4.1 S-Score（战略价值评分）

计算公式：
```
S = S1 × w1 + S2 × w2 + S3 × w3
```

| 组成 | 说明 | 数据来源 |
|------|------|----------|
| S1 | 客户等级评分 | customer_level |
| S2 | 毛利评分 | margin |
| S3 | 紧急度评分 | days_to_pdd |

### 4.2 P-Score（生产难度评分）

基于以下因素计算：
- 工艺难度
- 规格族系数
- 设备匹配度

### 4.3 综合优先级

计算公式：
```
Priority = ws × S + wp × P
```

其中 ws、wp 从 strategy_weights 表中根据策略名称动态加载。

### 4.4 人工调整

支持通过 Alpha 系数进行人工调整：
```
Final_Priority = Priority × (1 + Alpha)
```

## 6. 快捷操作指南

### 创建新策略
1. 进入 策略沙盘 → 策略模板
2. 点击预设模板卡片，或点击"创建新策略"按钮
3. 输入策略名称，调整 ws 和 wp 权重
4. 确保权重和为 1.0
5. 点击保存

### 模拟权重调整
1. 进入 策略沙盘 → 模拟计算
2. 选择基准策略
3. 调整 ws 和 wp 滑块
4. 可选：设置筛选条件
5. 点击"运行模拟"
6. 查看排名变化对比

### 对比两个策略
1. 进入 策略沙盘 → 结果对比
2. 选择策略 A 和策略 B
3. 点击"开始对比"
4. 查看排名差异统计和详细表格

### 人工调整优先级
1. 进入 合同管理 → 合同列表
2. 找到目标合同
3. 点击合同行展开详情
4. 设置 Alpha 系数
5. 填写调整原因
6. 确认保存
